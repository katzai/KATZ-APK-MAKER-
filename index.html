Creating a Distribution Certificate
01
Log in to developer.apple.com → Certificates, Identifiers & Profiles.
02
Click + → choose Apple Distribution → generate a CSR from Keychain Access on your Mac → upload → download the .cer.
03
Double-click the .cer to install in Keychain → right-click it → Export as .p12 → set a strong password → save.
04
Under Identifiers register your Bundle ID. Under Profiles create an Ad Hoc or App Store provisioning profile → download .mobileprovision.
05
Upload both files in iOS 02 → Signing. They are encrypted and stored as GitHub Actions secrets — the build repo is deleted after download.
⚠️ The p12 password must match the one you set during export. Mismatched passwords will cause the Actions build to fail at the "Import certificate" step.

Q
Build fails: "No signing identity found"?
Either upload p12 + mobileprovision in iOS 02, or leave both empty — the workflow auto-detects and builds unsigned.
Q
How many GitHub Actions minutes does macOS use?
macOS runners consume 10× the minutes of Linux. Free plan gives 2,000 Linux minutes = ~200 macOS minutes/month.
Q
Can I submit to the App Store?
Yes — build with a Distribution cert + App Store provisioning profile, then upload the IPA via Transporter or Xcode Organizer.
Q
WKWebView limitations?
Apple restricts JIT compilation in WKWebView for App Store apps. Most HTML/CSS/JS web apps work perfectly. Avoid heavy WASM JIT workloads.
Q
iPhone says "Untrusted Developer"?
Go to Settings → General → VPN & Device Management → tap your Apple ID → Trust. You only need to do this once per Apple ID per device.

GitHub Actions Workflow
①
KATZ generates a .github/workflows/build-apk.yml file that runs on every push to main.
②
The workflow: sets up JDK 17 → installs Gradle 8.2 → generates a keystore → runs assembleRelease → uploads the APK as an artifact.
③
KATZ polls the Actions API every 10 seconds and automatically downloads the APK when ready.

What You Download

📂

Common Errors
❌
"Invalid token — HTTP 401" — Token is wrong or expired. Regenerate at GitHub → Settings → Developer settings. Check you copied the full token.
❌
"All pushes failed — check token scope" — Your token is missing the repo scope. Create a new token with repo checked.
❌
Build failed / conclusion: failure — Check the Actions tab in your repo for detailed logs. Common causes: invalid package ID, bad Gradle setup, or incorrect source files.
❌
"Timed out waiting for build" — First build on a new account can take longer. Check Actions tab. If the build is still running, wait and download the artifact manually.
❌
Package ID invalid — Must be reverse-domain format: com.yourname.app. Use only lowercase letters, numbers, and dots. At least 2 segments separated by a dot.
