Identity
The display name shown on the home screen and app drawer. Keep it short (≤30 chars). Example: My App
Reverse-domain format. Must be unique on Play Store. Example: com.john.myapp. Use only lowercase letters, numbers and dots.
Human-readable version shown to users. Use semantic versioning: MAJOR.MINOR.PATCH e.g. 1.0.0
Internal integer version. Must increment with each update. Start at 1 and go up with each release.
Display

GitHub Token
How to create:
1. Go to GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. Click "Generate new token (classic)"
3. Check the repo scope (full repository access)
4. Copy the token (starts with ghp_)
⚠️ Token is used in your browser only and never stored.
🔑 Create token here → check repo scope → Generate Token.
Everything else is fully automatic: repo creation, secrets, workflow, APK download.

Creating a GitHub Token
01
Log in to github.com — create a free account if needed.
02
Click your profile picture → Settings → scroll to Developer settings (bottom left).
03
Click Personal access tokens → Tokens (classic) → Generate new token (classic).
04
Give it a name like katz-apk-maker. Set expiration to 90 days or No expiration.
05
Check the repo scope checkbox (this grants full repo access for creating repos and pushing files).
06
Click Generate token. Copy the token — it starts with ghp_. You won't see it again!
⚠️ The repo must have GitHub Actions enabled. Free accounts get 2,000 Actions minutes/month — more than enough for many builds.
GitHub Actions Workflow
①
KATZ generates a .github/workflows/build-apk.yml file that runs on every push to main.
②
The workflow: sets up JDK 17 → installs Gradle 8.2 → generates a keystore → runs assembleRelease → uploads the APK as an artifact.
③
KATZ polls the Actions API every 10 seconds and automatically downloads the APK when ready.
Supported Project Types
🌐
HTML/CSS/JS Web App — Upload HTML, CSS, JS files. Wrapped in a WebView. Perfect for single-page apps, games, or any web project.
🌐
Website URL — Enter a URL. The app opens that URL in a full-screen WebView. No files needed.
🐍
Python App — Upload Python files. Uses Chaquopy plugin to run native Python inside Android. Supports pip packages.
⚛️
React / React Native — Upload your built React files (after npm run build). For React Native, upload your source and a package.json.
🦋
Flutter — Upload your Flutter project files. The workflow runs flutter build apk --release.
⚙️
C / C++ — Upload your C/C++ source files. Uses Android NDK and CMakeLists.txt for native compilation.
☕
Java / Kotlin — Upload your Java or Kotlin source files. Standard Android Studio project structure expected.
🎮
Unity (WebGL) — Export your Unity project as WebGL build and upload the files. Wrapped in a WebView.
⚠️ Python, Flutter, C/C++ require more build time (~8–12 min). The app polls for up to 10 minutes.
What You Download
📦
GitHub delivers the APK as a ZIP archive. This is how GitHub Actions artifacts always work — it's a ZIP containing your .apk file.
📂
The ZIP is automatically downloaded to your browser's downloads folder. Open it and extract the .apk file inside.
Installing on Android
01
Extract ZIP — Right-click → Extract All (Windows) or double-click (Mac/Linux). Find the .apk file.
02
Transfer to phone — USB cable, Google Drive, WhatsApp "Send to yourself", or Bluetooth.
03
Enable Unknown Sources — Settings → Apps → Special access → Install unknown apps → Allow your file manager.
04
Install — Tap the APK file in your file manager → Install → Done! 🎉
Manual Download (if auto fails)
ℹ
Go to github.com/YOUR_USER/YOUR_REPO/actions → click the latest workflow run → scroll to Artifacts section → click the artifact to download.
✓ The APK is signed with a generated keystore. It installs and runs just like any APK.
Common Errors
❌
"Invalid token — HTTP 401" — Token is wrong or expired. Regenerate at GitHub → Settings → Developer settings. Check you copied the full token.
❌
"All pushes failed — check token scope" — Your token is missing the repo scope. Create a new token with repo checked.
❌
Build failed / conclusion: failure — Check the Actions tab in your repo for detailed logs. Common causes: invalid package ID, bad Gradle setup, or incorrect source files.
❌
"Timed out waiting for build" — First build on a new account can take longer. Check Actions tab. If the build is still running, wait and download the artifact manually.
❌
Package ID invalid — Must be reverse-domain format: com.yourname.app. Use only lowercase letters, numbers, and dots. At least 2 segments separated by a dot.
Tips
💡
Re-running a build? Use a different repo name or the tool will reuse the existing repo and update all files.
💡
Free GitHub accounts get 2,000 Actions minutes/month. Each build uses ~5–8 minutes. That's ~250–400 builds/month for free.
💡
